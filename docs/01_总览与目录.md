# 总览与目录

## 1. 这个仓库是给谁用的

主要给三类人用：

1. 继续维护当前课设工程的组员
2. 负责“速度估计方法本体”学习和改进的队友
3. 帮助队友做代码分析、生成补丁、整理结论的 agent

## 2. 先建立正确认识

当前项目分成两条主线：

- 主线 A：滤波实验主线  
  这条线已经能跑，已经有多种滤波方法、串口采集、HTML 对比和 MATLAB 仿真。

- 主线 B：速度估计方法本体研究线  
  这条线是新增分工，重点是把 `STO + PLL` 的代码层结构看懂，分析缺陷，尝试提出改进点。

两条线不是冲突关系，而是前后关系：

- A 线保证工程持续可运行、可展示、可答辩
- B 线帮助提升理论深度和代码层工作量

## 3. 目录怎么理解

### `firmware/current_project`

当前主工程，所有未来有效代码修改都应优先进入这里。

### `firmware/baseline_project`

基线工程，用于做 diff 和回退参考。  
它的价值不是运行，而是比较“当前工程到底改了哪些地方”。

### `patches/current_patch`

这里保留当前阶段的轻量补丁快照。适合：

- 快速 review
- 快速给 agent 看核心改动
- 做小范围代码阅读

但它不是权威工程，不能只改这里不改主工程。

### `tools/mc_speed_logger`

串口自动采集器。板上实验跑完后，会自动按规范命名保存 CSV。

### `tools/mc_speed_viewer`

本地 HTML 可视化页面。用于对比多个 CSV 的启动、稳态、调速和停转表现。

### `matlab/speed_filter_sim`

MATLAB 仿真目录。已经有：

- 总对比图
- 每种方法单图
- 指标 CSV

当前仿真是“滤波算法级仿真”，不是完整 FOC 闭环仿真。

### `docs/reference_notes`

精选资料和过程记录，已经把前期摸索、调参和路线调整保存下来，方便新成员补背景。

## 4. 当前建议阅读顺序

1. 先看根目录 `README.md`
2. 看 `docs/02_队友与Agent快速上手.md`
3. 看 `docs/03_代码入口与职责边界.md`
4. 如果负责速度估计任务，再看 `docs/04_速度估计任务学习与改进指南.md`
5. 开始改代码前，看 `docs/05_Git协作与PR流程.md`

## 5. 一句话提醒

这个仓库的真正目标不是“把文件都放进来”，而是让任何新成员拿到后都能回答：

- 当前工程用的是什么速度估计方法
- 当前核心改动在哪
- 如果我要改速度估计本体，应该从哪几份代码看起
- 我怎么提交 PR 才不会把主工程搞坏
