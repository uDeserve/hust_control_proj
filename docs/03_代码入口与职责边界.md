## 3. 当前速度链路怎么走

可以先按下面的简化链路理解：

```text
采样电流/电压
    ↓
STO + PLL 输出原始速度估计
    ↓
mc_tasks.c 中的自定义滤波层
    ↓
速度环使用滤波后的速度
    ↓
转矩/电流控制
    ↓
电机转速变化
```

## 4. 关键代码事实

当前主工程里已经能定位到这些事实：

- `STO_PLL_M1` 在 `mc_config.c` 里定义
- `STO_PLL_Init(&STO_PLL_M1)` 在 `mc_tasks.c` 中初始化
- `STO_PLL_CalcAvrgMecSpeedUnit(&STO_PLL_M1, &wAux)` 在运行期计算平均机械速度
- `MC_SPEED_FilterUpdateMecSpeed(&STO_PLL_M1._Super, wAux)` 在当前工程里对估计速度做滤波更新
- `STC_SetSpeedSensor(pSTC[M1], &SpeedFilterSensorM1)` 说明速度环最终吃的是滤波后的速度

## 5. 职责边界必须清楚

### 当前主工程维护者

职责：

- 保证主工程能继续正常编译和运行
- 管理滤波实验主线
- 决定哪些改动能合入

### 速度估计研究队友

职责：

- 读懂 `STO + PLL` 代码链
- 总结当前方法可能缺陷
- 提出尽量小、可验证、可回退的改动

### Agent

职责：

- 帮助定位代码
- 翻译原理
- 给出最小补丁建议
- 帮助生成对比分析文档

不应该由 agent 直接做的事：

- 在没明确边界的情况下大规模改控制链
- 一次性改多个模块再要求直接合并

## 6. 修改优先级建议

优先从下面顺序尝试：

1. 先只读代码，不改
2. 只加注释和文档
3. 只做离线分析或 MATLAB 回放
4. 对估计输出做小范围后处理
5. 再考虑触碰估计器本体
