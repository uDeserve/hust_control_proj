# 速度估计与滤波方法 资料导读

## 你的组选题需要解决什么

- 目标不是单纯让电机转起来，而是在现有 PMSM/FOC 框架里，围绕速度测量、速度估计或测速信号处理做一项可展示、可分析、可对比的改进。
- 这类题通常会落在三类工作上：
  1. 速度获取链路：编码器、Hall、反电势/观测器等。
  2. 速度估计链路：从位置、电流、电压或反电势推算机械角速度。
  3. 滤波链路：对测速值、估计值或控制反馈值做低通、均值、IIR/FIR、观测器平滑等处理。

## 做方案时重点看什么

- `11_STM32电动机控制应用系列讲座.md`
  作用：作为电机控制理论背景资料，优先用来补 FOC、速度环、观测器、反电势、位置/速度反馈的概念。
  限制：该 PDF 文本提取质量较差，适合知道“有这份资料”和“去原 PDF 翻对应页”，不适合直接把提取文本当讲义精读。

- `10_MCSDK5x_软件框架说明.md`
  作用：这份最关键。它告诉你 MCSDK 里和速度反馈直接相关的模块有哪些。
  已提取到的关键文件名包括：
  - `encoder_speed_pos_fdbk.c`
  - `hall_speed_pos_fdbk.c`
  - `speed_pos_fdbk.c`
  - `virtual_speed_sensor.c`
  - `speed_torq_ctrl.c`
  - `pid_regulator.c`
  判断：如果你们后面要改“速度估计与滤波方法”，大概率会落在这些模块附近。

- `14_MCSDK54_课件.md`
  作用：补充 Workbench 生成工程、CubeMX 配置边界、Motor Pilot 监控方式。
  关键点：不要随意改电机控制相关外设配置；生成工程后，算法改动应优先放在用户可控或库扩展位置。

## 做实现时重点看什么

- `12_STM32G4_电机控制外设.md`
  作用：看采样、定时器、编码器接口、ADC 触发等。
  限制：提取文本质量一般，但至少能确认这份资料覆盖 timer / encoder / ADC。
  用法：需要看定时器同步、编码器接口、ADC 采样触发关系时，回原 PDF。

- `13_STM32G4_特别外设助力电机控制.md`
  作用：这是你们组选题最可能做出亮点的一份资料。
  直接有用的点：
  - `CORDIC`：可用于三角函数、反正切、模值等运算加速，适合观测器、角度/速度相关计算。
  - `FMAC`：硬件滤波加速器，明确支持 `FIR`、`IIR`，非常适合做“速度滤波方法”的实现或优化。
  - `OPAMP`/`ADC`：影响电流采样质量，也会间接影响估计器输入质量。
  判断：如果你们想把课设做得像“方法 + 工程实现”，`FMAC 做滤波` 是很值得优先考虑的方向。

- `30_STM32G474RE_数据手册.md`
  作用：确认芯片是否真的支持你想用的外设。
  已确认的关键资源：
  - `CORDIC`
  - `FMAC`
  - `5 x 12-bit ADC`
  - 多个 `timer`
  - `quadrature encoder input`
  判断：从芯片能力上看，G474 很适合做测速与滤波实验。

- `31_STM32G4_参考手册.md`
  作用：落地查寄存器和章节入口。
  已定位的高价值章节：
  - `17 CORDIC co-processor`
  - `18 Filter math accelerator (FMAC)`
  - `18.3.8 Implementing FIR filters with the FMAC`
  - `18.3.9 Implementing IIR filters with the FMAC`
  - `11.3.2 From timer (TIMx, HRTIM) and EXTI to ADC (ADCx)`
  - `7.2.11 ADC clock`
  - `7.2.13 Timer clock`
  判断：后续只要决定做 FIR/IIR/测速链路优化，这份手册就会成为主查文档。

## 现在可以考虑的实现方向

- 方向 A：编码器/Hall 速度值数字滤波
  特点：实现难度低，容易出对比图，适合课设节奏。
  可做内容：滑动平均、一阶低通、IIR/FIR，对比滤波前后速度纹波、响应速度和控制稳定性。

- 方向 B：基于 MCSDK 速度反馈模块做估计值平滑
  特点：更贴近软件框架本体，适合做“在现有库上改进”。
  可做内容：在 `speed_pos_fdbk` 链路加入滤波或估计改进。

- 方向 C：利用 `FMAC` 做硬件加速滤波
  特点：最有工程特色，汇报也更容易讲出亮点。
  风险：实现复杂度更高，需要读 `RM0440` 的 FMAC 章节。

- 方向 D：结合 `CORDIC` 做角度/反三角运算加速
  特点：偏“估计器计算优化”。
  风险：需要你们具体方案里真的存在这类运算，否则会显得为硬件而硬件。

## 当前建议

- 第一优先：先把课设目标和实验平台跑通，再决定是做“滤波”还是做“估计”。
- 第二优先：从 `MCSDK5x_软件框架说明` 里找速度反馈相关代码入口。
- 第三优先：若老师允许做工程优化型选题，优先考虑 `FMAC + 速度滤波`，这是最容易兼顾“理论、实现、展示”的方案。
