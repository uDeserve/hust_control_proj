# 速度估计与滤波 后续工作指南

## 1. 先把题目理解定住

结合老师/助教的回复，目前比较稳妥的理解是：

- 课设最基本的要求，是把题目本身要求完成。
- 额外做出来的内容，主要用于体现工作量、体现对电机控制的理解，从而争取更高分。
- 因此当前最优策略不是马上重写一套新的速度估计算法，而是先把“现有速度估计方法说明 + 滤波实现 + 结果回显与对比”做完整。
- 老师后续反馈强调“先仿真，再实操，再复盘，再调优”，所以后续所有滤波和参数调整都要先补 MATLAB 仿真，再做板上实验。

这条路线的好处：

- 风险低，能稳稳把基础要求完成。
- 当前工程已经跑通，实验链路已经建立起来。
- 现阶段继续深挖滤波、指标分析、展示方式，性价比最高。

## 2. 当前工程到底在做什么

当前工程默认不是 Hall，也不是编码器，而是无感速度估计链路。

关键信息如下：

- `first_try.wb_def` 里：
  - `STATE_OBSERVER_PLL = TRUE`
  - `SPEED_SENSOR_SELECTION = STATE_OBSERVER_PLL`
  - `HALL_SENSORS_AVAILABLE = FALSE`
  - `ENCODER_AVAILABLE = FALSE`
- `mc_config.c` 里已经生成了 `STO_PLL_Handle_t STO_PLL_M1`
- `mc_tasks.c` 运行时调用了 `STO_PLL_CalcAvrgMecSpeedUnit()`

因此，后续汇报里可以明确写：

- 本系统原始速度估计方法采用 `State Observer + PLL`
- 我们当前的主要工作，是在该估计结果基础上做速度滤波、结果对比和实验分析

这个表述是符合当前工程真实情况的，也最容易讲清楚。

## 3. 当前阶段的总体路线

后续工作分三层推进：

### 第一层：理解现有速度估计方法

要完成的内容：

1. 说明当前工程使用 `State Observer + PLL`
2. 说明它不是 Hall/编码器测速
3. 说明速度估计值可能存在噪声、纹波、低速误差和动态滞后
4. 说明为什么估计速度进入速度环前需要滤波

这部分是必须完成的主体。

### 第二层：补 MATLAB 仿真

后续算法调整不要直接上板硬试。先在 MATLAB 里做简化仿真：

- 目标速度：`0 -> 500 rpm`
- 调速曲线：`0 -> 500 -> 1000 -> 500 -> stop`
- 原始估计速度：真实速度叠加噪声、周期纹波和动态扰动
- 滤波方法：`NONE`、`LPF1`、`MOVAVG8`、`WMA4`、`ADALPF`
- 输出：曲线图和指标表

这一步的目标不是完整复现 MCSDK 的底层观测器，而是先验证滤波策略的趋势。

### 第三层：把实测分析做扎实

不能只停留在“曲线看起来有差别”，要补充量化指标，并和仿真结果对照。

建议至少统计：

- 上升时间
- 进入稳态时间
- 超调量
- 稳态平均值
- 稳态波动
- 相对目标转速的平均误差

这部分做出来以后，PPT 的说服力会明显提升。

### 第三层：补一个高分亮点

建议优先级如下：

1. 自适应滤波
2. FMAC 硬件滤波
3. 现有估计器参数整定对比
4. 新的速度估计方法

原因：

- 自适应滤波最容易在当前工程基础上做出明显效果。
- FMAC 很有工程特色，适合讲“硬件加速滤波”。
- 直接重写新的速度估计方法，工作量和调试风险都很高，现阶段不建议作为第一优先级。

## 5. 接下来建议做的具体事情

### 5.1 第一优先：补 MATLAB 仿真

已经新增仿真目录：

- `matlab_speed_filter_sim/speed_filter_sim_main.m`
- `matlab_speed_filter_sim/README.md`

需要先在 MATLAB 中运行脚本，导出两类结果：

- 主实验：`0 -> 500 rpm`
- 调速实验：`0 -> 500 -> 1000 -> 500 -> stop`

仿真结果用于指导后续板上实验，也用于 PPT 展示“先仿真、再实测”的完整流程。

### 5.2 第二优先：补实验分析文档

需要输出一份规范化实验记录，内容包括：

- 每种滤波方法的原理
- 参数设置
- MATLAB 仿真曲线和指标
- 实验条件
- 板上实测曲线
- 指标统计表
- 仿真和实测差异分析
- 结论和下一轮调参方向

这一份文档将来既可以服务 PPT，也可以服务答辩。

### 5.3 第三优先：把 CSV 分析自动化

当前已经能看图，但还需要进一步做自动统计。

建议把以下统计流程固定下来：

- 自动读取指定实验 CSV
- 自动识别启动段、稳态段、停止前段
- 自动计算关键指标
- 自动输出总表

这样以后每改一次滤波，都能快速重新出结果。

### 5.4 第四优先：做一个更像“自己思考过”的滤波方案

最推荐的是自适应滤波：

- 启动阶段：弱滤波，保证响应快
- 稳态阶段：强滤波，减小波动

这个思路很适合你们现在的题目，因为它能同时解释：

- 为什么固定强滤波会拖慢响应
- 为什么固定弱滤波稳态波动较大
- 为什么切换策略更合理

如果这个效果做出来，答辩时非常好讲。

### 5.5 第五优先：再决定要不要碰新的速度估计方法

只有在前面三层都比较稳了之后，再考虑是否进一步做：

- `STO + PLL` 参数整定实验
- `STO + CORDIC` 可行性调研
- 其他估计方法的理论对比

这里先不承诺一定实现，只保留为进阶方向。

## 6. 当前不建议优先做的事情

以下事项暂时不建议作为第一优先级：

### 6.1 不建议现在从零重写新的速度估计器

原因：

- 当前题目基础要求还可以继续做扎实
- 新估计器实现难度大
- 调不稳的概率高
- 时间花得多，但未必比“实验分析 + 优化展示”更容易转化成分数

### 6.2 不建议把主要精力花在 Motor Pilot 显示方式上

原因：

- `Motor Pilot` 更像调试工具，不是课设本体
- 你们已经有自己的 CSV 可视化页面
- 自己的页面反而更适合做多方法对比和结果汇总

## 9. 一句话结论

当前最合理的路线是：

- 不急着重写新的速度估计方法
- 先把现有 `STO + PLL` 讲清楚
- 先用 MATLAB 仿真验证滤波趋势
- 把滤波实现、数据采集、对比分析和展示做完整
- 再补一个高性价比创新点冲高分

这条路线最稳，也最符合目前工程进度。
