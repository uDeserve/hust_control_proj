# 速度估计与滤波 实现说明

## 1. 现在这套工程的速度估计到底是什么

先把一个最容易混淆的点说清楚：

- 当前工程的原始速度估计，不是我们自己从零写的
- 它来自 `MCSDK` 生成工程里已经选好的无感方案
- 当前配置实际使用的是 `State Observer + PLL`

也就是说，我们现在做的工作不是“替换掉底层估计器”，而是：

1. 把现有速度估计链路说明清楚
2. 在估计结果基础上实现多种速度滤波
3. 做实验采集、可视化和对比分析

这个路线和老师/助教的回复是一致的，也最适合当前阶段冲高分。

## 2. 代码里哪些是现有估计，哪些是我们补的

### 2.1 现有估计链路

工程配置里已经指定：

- `STATE_OBSERVER_PLL = TRUE`
- `SPEED_SENSOR_SELECTION = STATE_OBSERVER_PLL`

对应到运行时代码里，核心基线是 `STO + PLL`。

### 2.2 我们补的实验和滤波逻辑

本次主要新增逻辑在：

- [mc_tasks.h](/mnt/e/hermes/firmware_exp_patch/Inc/mc_tasks.h:51)
- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:48)

关键思路是：

- 先拿到底层观测器输出的原始机械速度
- 再做我们自己的滤波
- 然后把滤波后的速度重新送回速度环
- 同时通过串口输出 `CSV`，方便本地分析

这意味着我们现在的实验结果不是“离线假分析”，而是滤波确实进入了控制闭环。

## 3. 现在已经实现的滤波方法

当前实验配置表在：

- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:148)

目前一共 7 组：

1. `NONE`
2. `LPF1 shift2`
3. `LPF1 shift3`
4. `LPF1 shift4`
5. `MOVAVG8`
6. `WMA4`
7. `ADALPF 2to4_err35`

下面分别说明。

## 4. 各滤波方法解释

### 4.1 NONE

公式理解：

- 不做额外滤波
- 直接把观测器输出当作速度环输入

特点：

- 响应最快
- 抖动也最明显
- 适合作为基线对照组

### 4.2 一阶低通 `LPF1`

代码位置：

- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:1275)

离散形式可以理解为：

```text
y(k) = y(k-1) + (x(k) - y(k-1)) / 2^shift
```

其中：

- `x(k)` 是原始估计速度
- `y(k)` 是滤波后速度
- `shift` 越大，滤波越强

特点：

- 算法最简单
- 资源开销很小
- 很适合 MCU 实时跑
- 但 `shift` 太大时会出现明显滞后

实验里我们保留了三档：

- `shift2`：响应较快，平滑一般
- `shift3`：通常是比较均衡的一档
- `shift4`：更平滑，但往往会拖慢动态

### 4.3 普通滑动平均 `MOVAVG8`

代码位置：

- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:1239)

原理：

- 保存最近 `8` 个点
- 直接求平均

公式：

```text
y(k) = (x(k) + x(k-1) + ... + x(k-7)) / 8
```

特点：

- 抖动压制比较明显
- 但窗口越长，启动响应越慢
- 容易让停止和加减速边缘变钝

### 4.4 加权滑动平均 `WMA4`

代码位置：

- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:1252)

原理：

- 也是看最近几个点
- 但不是每个点权重一样
- 越新的点权重越大

本实验使用：

```text
y(k) = [1*x(k-3) + 2*x(k-2) + 3*x(k-1) + 4*x(k)] / 10
```

特点：

- 相比普通平均，更重视当前变化
- 通常比 `MOVAVG8` 响应更快
- 同时比纯 `LPF1 shift2` 更平滑

这个方法很适合答辩时讲“不是所有平均都一样”。

### 4.5 自适应一阶低通 `ADALPF`

代码位置：

- [mc_tasks.c](/mnt/e/hermes/firmware_exp_patch/Src/mc_tasks.c:1276)

本实验使用的策略是：

- 当 `|目标速度 - 当前原始速度| >= 35 rpm` 时，用 `shift=2`
- 当误差较小进入稳态后，用 `shift=4`

也就是：

- 启动和大扰动阶段：弱滤波，保证反应快
- 稳态阶段：强滤波，减小波动

特点：

- 很符合工程直觉
- 比固定参数滤波更容易讲出“为什么这样设计”
- 如果实验效果稳定，这一项会是比较好的高分点

不过从现在这一轮结果看，`ADALPF` 还没有明显跑赢 `LPF1 shift3`，所以后续建议先做一轮参数寻优，再进入重复性测试。
推荐优先调的维度是：

- `enter_rpm`：进入强滤波的门限
- `exit_rpm`：退出强滤波的门限
- `confirm_n`：稳态确认次数
- `fast_shift / slow_shift`：快慢滤波档位

## 5. 为什么这些方法适合现在的课设

因为这个课设现在最需要的是三件事：

1. 把“速度估计”这件事讲清楚
2. 把“速度滤波实现”做实
3. 把“结果回显和对比”做完整

这些滤波方法正好对应不同层次：

- `LPF1`：基础方法
- `MOVAVG8`：经典平均法
- `WMA4`：比普通平均更进一步
- `ADALPF`：带工程思考的改进方法

这样汇报时就不是“堆代码”，而是有层次地推进。

## 6. CSV 采集和页面分析怎么看

当前 `CSV` 里主要列有：

- `time_ms`
- `target_speed_rpm`
- `raw_speed_rpm`
- `filtered_speed_rpm`
- `final_speed_rpm`
- `method_name`
- `param_tag`

页面位置：

- [index.html](/mnt/e/hermes/mc_speed_viewer/index.html:1)
- [app.js](/mnt/e/hermes/mc_speed_viewer/app.js:1)

现在页面除了原来的曲线显示，还新增了“实验指标对比”表，主要自动算：

- 上升时间
- 稳态建立时间
- 稳态均值
- 稳态误差
- 稳态标准差
- 稳态峰峰值
- 最大超调
- 总时长

这样以后每次重新采集一组 `CSV`，都能直接出对比结果。

## 7. 后续最值得继续做的事情

建议优先级：

1. 先把 `ADALPF` 再调几轮，筛出更像样的一组参数
2. 再对主实验和调速实验做重复测试
3. 用页面导入新的 `CSV`，输出指标表
4. 对比 `LPF1 shift3 / WMA4 / ADALPF`
5. 固化一版实验结论
6. 再决定要不要继续做更复杂的创新点

当前这版实现的定位很明确：

- 保留工程原有 `STO + PLL` 速度估计作为基线
- 在其基础上实现多种滤波方法
- 通过串口采集和本地页面完成结果回显与对比

这已经能比较完整地覆盖题目要求，而且有继续往高分方向延展的空间。
