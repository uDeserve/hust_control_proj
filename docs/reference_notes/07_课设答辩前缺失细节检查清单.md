# 课设答辩前缺失细节检查清单

## 必须补

1. 补 MATLAB 仿真
- 主实验：`0 -> 500 rpm`
- 调速实验：`0 -> 500 -> 1000 -> 500 -> stop`
- 仿真里要包含原始估计速度噪声、滤波输出和指标表
- 汇报时要说明仿真用于验证滤波趋势，不是假装完整复现 MCSDK

1. 主实验和调速实验分开写清楚
- 主实验：`0 -> 500 rpm`
- 调速实验：`0 -> 500 -> 1000 -> 500 -> stop`
- 两者用途不要混在一起

2. 指标定义写明确
- 上升时间怎么取
- 稳态区间怎么取
- 超调量怎么算
- 停转时间从哪里开始算

3. 控制变量写明确
- 同一电机、同一供电、同一硬件环境
- 每组实验是否都先停稳再开始
- 采集条件是否一致

4. 说明“滤波为什么会影响控制”
- 滤波后的速度会回送到速度环
- 所以会影响启动、稳态和停转

5. 统一答辩口径
- 基线方法：`STO + PLL`
- 重点工作：速度滤波 + 回显 + 对比
- 创新点：自适应滤波或参数整定
- 总体流程：理解估计方法 -> MATLAB 仿真 -> 固件实现 -> 板上实验 -> 数据复盘

6. 讲清楚速度估计部分
- 当前工程不是 Hall，也不是编码器
- 当前速度估计由 MCSDK 生成的 `STO + PLL` 完成
- 我们没有重写底层估计器，而是在估计结果上做滤波改进
- 估计速度存在噪声和动态误差，所以需要滤波

## 建议补

1. 做重复性实验
- 至少对 `NONE`、`LPF1 shift3`、`MOVAVG8/WMA4` 做重复测试
- 给出均值和标准差
- `ADALPF` 先完成参数寻优，再进入重复测试

2. 补参数选择依据
- 为什么选 `shift2/3/4`
- 为什么选 `MOVAVG8`
- 为什么选 `WMA4`
- 为什么选当前自适应阈值

3. 补失败过程记录
- 早期串口卡住的问题
- 早期 `session_end` 重复刷屏的问题
- 早期自适应参数效果不佳的问题

4. 补整体链路图
- `STO+PLL -> 速度滤波 -> 速度环 -> 电机`
- `UART CSV -> Logger -> HTML Viewer`
- `MATLAB 仿真 -> 固件实现 -> 实测 CSV -> 复盘调参`

5. 补硬件和实验环境说明
- 板卡型号
- 电机型号
- 串口号
- `USER` 键操作流程

6. 补方法优劣总结
- 哪个最稳
- 哪个最平滑
- 哪个响应最快
- 哪个最适合当前课设

## 可选加分

1. 导出更多控制量
- 比如速度环输出量
- 更容易说明“滤波影响了控制”

2. 做局部放大图
- 重点看加速、减速、停转瞬间

3. 做局限性说明
- 空载/轻载条件
- 自适应参数仍可继续优化

4. 固化成一页答辩口径
- 一句话介绍题目
- 一句话介绍方案
- 一句话介绍结果
- 一句话介绍创新点

5. 答辩前代码讲解训练
- 能指出 `mc_tasks.c` 里实验配置表的位置
- 能解释 `raw_speed_rpm`、`filtered_speed_rpm`、`final_speed_rpm`
- 能解释 USER 键如何启动自动实验
- 能解释 ADALPF 参数 `2to4_h18_40_n32` 的含义

## 当前优先级

1. 先补 MATLAB 仿真
2. 再补实验设计和指标定义
3. 再把 `ADALPF` 最优参数带入正式实验
4. 再补重复性实验和参数依据
5. 再补链路图和失败过程
6. 最后做答辩代码讲解训练
