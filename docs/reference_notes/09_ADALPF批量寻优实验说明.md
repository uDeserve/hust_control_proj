# ADALPF 批量寻优实验说明

## 目的

一次自动跑多组 `ADALPF` 参数，先筛出更好的门限组合，再进入重复性测试。

## 当前实现

- 已在固件里加了批量寻优模式
- 当前每组都跑同一套阶跃实验
- 每组参数会自动写入 `param_tag`
- 每组结果会单独生成 CSV

## 批量参数

当前建议先跑 9 组：

- `enter_rpm = 12 / 18 / 24`
- `exit_rpm = 40`
- `confirm_n = 32 / 64 / 96`

组合如下：

- `2to4_h12_40_n32`
- `2to4_h12_40_n64`
- `2to4_h12_40_n96`
- `2to4_h18_40_n32`
- `2to4_h18_40_n64`
- `2to4_h18_40_n96`
- `2to4_h24_40_n32`
- `2to4_h24_40_n64`
- `2to4_h24_40_n96`

## 评价思路

跑完后先看这几项：

1. 启动是否拖慢
2. `1000 rpm` 段是否更稳
3. `500 rpm` 回切是否更平滑
4. 停转尾段是否更短

## 结果筛选

先从 9 组里挑出 1~2 组：

- 启动不肉
- 稳态不抖
- 停转不过分拖尾

然后再对这 1~2 组做重复性实验。

## 后续顺序

1. 批量寻优
2. 选出最优参数
3. 重复性测试
4. 输出最终结论
