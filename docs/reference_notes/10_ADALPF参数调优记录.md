# ADALPF 参数调优记录

## 1. 调优目的

先用 `0 -> 500 rpm` 阶跃实验筛出更合适的自适应滤波参数，再把最优候选拿去做后续完整调速实验和重复性测试。

## 2. 本轮参数组

本轮一次跑了 9 组：

- `2to4_h12_40_n32`
- `2to4_h12_40_n64`
- `2to4_h12_40_n96`
- `2to4_h18_40_n32`
- `2to4_h18_40_n64`
- `2to4_h18_40_n96`
- `2to4_h24_40_n32`
- `2to4_h24_40_n64`
- `2to4_h24_40_n96`

## 3. 评价口径

主要看这几项：

- 启动峰值是否过高
- 超调是否过大
- 停转尾段是否拖得太长
- 稳态尾段是否抖动明显

## 4. 调优结果

| 参数组 | 峰值 | 超调 | 停转<=30rpm | 尾段均值±std |
|---|---:|---:|---:|---:|
| `2to4_h12_40_n32` | 684 | 184 | 480 ms | 498.0 ± 0.0 |
| `2to4_h12_40_n64` | 684 | 184 | 480 ms | 496.8 ± 2.4 |
| `2to4_h12_40_n96` | 660 | 160 | 480 ms | 499.2 ± 5.9 |
| `2to4_h18_40_n32` | 612 | 112 | 480 ms | 498.0 ± 8.5 |
| `2to4_h18_40_n64` | 672 | 172 | 480 ms | 496.2 ± 3.8 |
| `2to4_h18_40_n96` | 702 | 202 | 480 ms | 498.0 ± 0.0 |
| `2to4_h24_40_n32` | 660 | 160 | 480 ms | 499.2 ± 5.9 |
| `2to4_h24_40_n64` | 696 | 196 | 480 ms | 496.2 ± 5.4 |
| `2to4_h24_40_n96` | 648 | 148 | 480 ms | 496.8 ± 2.4 |

## 5. 结论

1. 这轮里 `2to4_h18_40_n32` 的综合表现最好，峰值最低，超调也最小。
2. `h18_40` 这一档整体比 `h12_40` 和 `h24_40` 更平衡。
3. `confirm_n` 不是越大越好，`32` 比 `64 / 96` 更容易压住超调。
4. 这轮 `ADALPF` 还没到最终定版，只是筛出了更值得保留的候选。
