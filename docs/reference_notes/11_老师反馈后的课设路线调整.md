# 老师反馈后的课设路线调整

## 1. 当前判断

老师这次反馈说明，当前路线不用推翻，但需要重新包装得更完整。

原来我们主要在做：

- 基于 MCSDK 自动生成的速度估计结果
- 增加多种速度滤波方法
- 采集 CSV 并做本地 HTML 可视化对比
- 通过参数寻优改进自适应滤波效果

这个方向是成立的，因为题目里的“速度估计与滤波”不一定要求从零重写底层估计器。现有工程已经使用了 MCSDK 的无感速度估计方法，我们可以把它作为基线，然后围绕它做分析和改进。

但是现在必须补强三件事：

1. 讲清楚当前速度估计方法是什么。
2. 在上板实验前补 MATLAB 仿真。
3. 把“滤波为什么影响控制效果”讲成闭环逻辑，而不是只讲图像好不好看。

## 2. 题目口径应该怎么说

推荐后续统一表述为：

> 本课设以 MCSDK 生成工程中的 `State Observer + PLL` 无感速度估计为基线，研究估计速度信号的滤波处理方法。我们在理解原始速度估计链路的基础上，实现一阶低通、滑动平均、加权滑动平均和自适应低通等滤波方法，并通过 MATLAB 仿真、板上实验、CSV 数据采集和 HTML 可视化对比，分析不同滤波方法对启动响应、稳态波动、调速过程和停转过程的影响。

这句话里包含了老师关心的四点：

- 速度估计不是没做，而是用了现有 MCSDK 基线并进行理解分析。
- 滤波是我们主要实现和改进的部分。
- 有仿真，不是直接上板硬试。
- 有实测和对比，不是只停留在理论。

## 3. 为什么暂时不重写新的速度估计器

不建议当前阶段从零写新的速度估计器，原因如下：

- `STO + PLL` 本身已经是 MCSDK 提供的成熟无感估计方案。
- 新估计器会涉及电机模型、电流电压采样、反电势估算、角度锁相、启动过渡等底层逻辑。
- 如果改动底层估计器，容易影响 FOC 主链路，调不稳的风险明显高于滤波改进。
- 课设评分更看重工作量、理解和实验闭环，而不是一定要重写工业级估计器。

因此当前更合理的方案是：

1. 把现有 `STO + PLL` 原理讲清楚。
2. 分析估计速度可能存在噪声、纹波、动态超调和稳态抖动。
3. 用滤波方法改进速度反馈质量。
4. 用仿真和实测证明改进确实有效。

这个路线既真实，又能体现工作量。

## 4. 后续标准流程

后续所有算法改动都按下面流程走：

1. 理论理解：先说明原方法和问题。
2. 仿真验证：在 MATLAB 中构造目标速度、估计速度噪声和滤波算法，先看趋势是否合理。
3. 固件实现：把仿真中表现较好的方法写进 `mc_tasks.c`。
4. 板上实验：用 USER 键自动跑固定流程，串口输出 CSV。
5. 数据分析：用 HTML 页面和指标表比较响应速度、超调、稳态波动、停转时间。
6. 复盘调参：根据实测结果调整参数，再进入下一轮。

这条链路可以在 PPT 里画成一张流程图：

```text
STO+PLL 速度估计理解
        ↓
估计速度噪声/纹波问题分析
        ↓
滤波算法设计
        ↓
MATLAB 仿真验证
        ↓
MCSDK 工程代码实现
        ↓
电机实测 CSV 采集
        ↓
HTML 可视化 + 指标统计
        ↓
参数调优和结论
```

## 5. 速度估计部分要做到什么程度

最低要能讲清楚：

- 当前工程为什么不是 Hall/编码器测速。
- 当前工程使用的是 `State Observer + PLL`。
- `State Observer` 大致根据电机模型、电压电流和反电势估计转子相关信息。
- `PLL` 的作用是让估计角度跟随观测到的电角度，并从角度变化中得到速度。
- 速度估计值不是完美真实值，会受到采样噪声、电机参数误差、负载变化、低速反电势弱等因素影响。
- 因此速度估计结果进入速度环之前，有必要做滤波和平滑。

暂时不要求做到：

- 自己完整推导并重写 STO。
- 改写 MCSDK 底层观测器。
- 替换成另一种无感估计方法。

如果答辩老师问为什么没重写估计器，可以回答：

> 我们这次把 MCSDK 的 `STO + PLL` 作为工业基线方法，重点研究估计速度信号进入速度环前的滤波处理。底层估计器重写风险较高，容易影响 FOC 主链路稳定性，所以我们选择在现有估计链路上做可验证的滤波改进，并通过仿真和实测评价效果。

## 6. 滤波为什么会影响启动和稳态

我们当前代码不是只把滤波结果拿来显示，而是把滤波后的速度送回速度环。

简化链路是：

```text
电流/电压采样
    ↓
STO+PLL 原始速度估计 raw_speed
    ↓
我们新增的速度滤波 filtered_speed
    ↓
速度环 PI 控制
    ↓
电流给定 / 转矩输出
    ↓
电机转速变化
```

所以滤波会改变控制器看到的“当前速度”：

- 滤波太弱：速度反馈抖动大，速度环输出也更容易抖。
- 滤波太强：反馈滞后，控制器以为速度还没到，可能导致启动超调或调速反应慢。
- 自适应滤波：动态阶段少滤一点，稳态阶段多滤一点，目标是兼顾响应速度和稳态平滑。

这也是我们做多方法对比和参数寻优的核心原因。

## 7. 仿真要补什么

MATLAB 仿真不需要完整复现 MCSDK 的 FOC 和 STO。当前阶段仿真的目标是验证滤波策略，所以模型可以简化为：

- 目标速度：`0 -> 500 rpm` 阶跃，或者 `0 -> 500 -> 1000 -> 500 -> stop` 调速曲线。
- 真实速度：用一阶惯性模型模拟电机速度跟随。
- 原始估计速度：在真实速度上叠加随机噪声、周期纹波和动态扰动。
- 滤波输出：分别套用 `NONE`、`LPF1`、`MOVAVG8`、`WMA4`、`ADALPF`。
- 指标评价：上升时间、超调、稳态误差、稳态标准差、调速段滞后。

这样做的意义是：

- 先在仿真里看各滤波方法的理论趋势。
- 再用实测 CSV 检查真实电机是否符合这个趋势。
- 如果仿真和实测有差异，反而可以作为复盘内容，比如真实系统存在启动非线性、估计器低速误差、串口采样周期等影响。

## 9. 近期工作优先级

当前按下面顺序推进：

1. 补 MATLAB 滤波仿真，先生成可展示曲线和指标。
2. 根据仿真结果确认 ADALPF 参数是否还需要调整。
3. 把固件从 ADALPF 批量寻优模式恢复成正式对比实验模式。
4. 保留两类实验：主实验 `0 -> 500 rpm`，调速实验 `0 -> 500 -> 1000 -> 500 -> stop`。
5. 做正式多组重复实验，统计均值和标准差。
6. 整理报告/PPT：理论、仿真、实测、复盘、创新点。
